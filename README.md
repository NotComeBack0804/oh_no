# Oh No - AI 协作开发团队

> 三个 AI Agent 协同工作，完成产品从需求到上线的高质量交付。

## 👥 Agent 团队

| Agent | 角色 | 职责 |
|--------|------|------|
| 📋 **PM** | 资深产品经理 | 需求分析、任务拆解、进度协同 |
| 👨‍💻 **Engineer** | 全栈研发工程师 | 技术方案设计、代码开发、Demo 交付 |
| 🔍 **Tester** | 质量保障测试工程师 | 接口测试、安全审计、测试报告 |

## 🔄 协作工作流

```
┌─────────────────────────────────────────────────────────────────┐
│                         启动阶段                                  │
│  用户输入原始需求 → PM 输出《需求拆解任务列表》                      │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                         开发阶段                                  │
│  PM @工程师 → 工程师接收任务 → 开发 Demo + 接口文档 → @测试工程师    │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                         测试阶段                                  │
│  测试工程师执行测试 → 输出《测试报告》                              │
│       ↓                                                          │
│  ┌──────────────┐    ┌──────────────┐                           │
│  │ 通过 → 通知PM验收 │  │ 不通过 → 打回研发修复 │                    │
│  └──────────────┘    └──────────────┘                           │
└─────────────────────────────────────────────────────────────────┘
```

## 📌 输出物清单

| 角色 | 输出物 |
|------|--------|
| PM | 《需求拆解任务列表》、任务卡片 |
| Engineer | 代码工程、API接口文档、Demo |
| Tester | 测试用例、安全审计报告、测试报告 |

## 🚀 使用方式

**当你有项目需求时：**

1. 将需求文档或描述发送给我
2. 我会让 **PM Agent** 分析需求 → 拆解任务 → 分发给工程师
3. **Engineer Agent** 接收任务 → 开发 Demo → 输出接口文档 → 提测
4. **Tester Agent** 执行测试 → 输出测试报告
5. 通过则通知 PM 验收；不通过则打回工程师修复
