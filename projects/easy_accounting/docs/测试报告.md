### 7.4 发现的问题

#### 🔴 Bug 1 [P1] - `currentTime` 未定义（编译错误）

**文件**: `PayAccessibilityService.kt`  
**位置**: `handleWindowStateChanged()` 第 ~92 行，`handleWindowContentChanged()` 第 ~107 行

**问题描述**:  
在这两个方法中，代码引用了变量 `currentTime`，但该变量未在方法作用域内定义。

```kotlin
// handleWindowStateChanged() 内：
if (isPaymentPage) {
    lastProcessedTime = currentTime  // ❌ currentTime 未定义！
    lastProcessedText = text
    extractAndSavePayment(packageName, text)
}

// handleWindowContentChanged() 内：
if (isPaymentPage) {
    lastProcessedTime = currentTime  // ❌ currentTime 未定义！
    lastProcessedText = text
    extractAndSavePayment(packageName, text)
}
```

`currentTime` 仅在 `onAccessibilityEvent()` 内作为局部变量定义，不在 `handleWindowStateChanged()` 和 `handleWindowContentChanged()` 的作用域内。

**影响**: 该 Bug 会导致**编译失败**，自动抓取功能完全不可用。

**修复建议**:
```kotlin
// 方案1：在每个方法内直接使用 System.currentTimeMillis()
if (isPaymentPage) {
    lastProcessedTime = System.currentTimeMillis()
    ...
}

// 方案2（推荐）：在类级别添加 lastProcessedTimeMillis 变量，
// 在 onAccessibilityEvent() 计算后传递给两个方法
private fun handleWindowStateChanged(event: AccessibilityEvent, ..., currentTime: Long) {
    if (isPaymentPage) {
        lastProcessedTime = currentTime
        ...
    }
}
```

**严重程度**: 🔴 P1（编译阻塞）

#### 🟡 问题 2 [P2] - confirmBill 中 accountId 硬编码

**文件**: `FloatBubbleService.kt` 第 ~163 行

```kotlin
val bill = Bill(
    ...
    accountId = if (record.source == PaySource.ALIPAY) 1L else 2L, // ❌ 硬编码账户ID
    ...
)
```

**问题描述**:  
账户ID直接硬编码为 `1L`（支付宝）和 `2L`（微信），依赖数据库初始化时的插入顺序。如果账户表数据被手动修改或重建，可能导致账单记入错误的账户。

**影响**: 金额可能记入错误的账户，但数据库初始化后通常能正常工作。

**修复建议**:  
使用 `AccountType` 枚举查询对应账户的实际 ID，而非硬编码：
```kotlin
val accountId = when (record.source) {
    PaySource.ALIPAY -> database.accountDao().getByType(AccountType.ALIPAY)?.id ?: 1L
    PaySource.WECHAT -> database.accountDao().getByType(AccountType.WECHAT)?.id ?: 2L
}
```
